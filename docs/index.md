## Getting Started

### Development

```bash
# Install dependencies
npm install

# Start development server (includes content build)
npm run dev
```

### Production Build

```bash
# Build for production
npm run build

# Preview production build
npm run preview
```

### Adding Content

1. **Blog Post:** Create `content/blog/my-post.md` with YAML front matter
2. **Project:** Create `content/projects/my-project.md` with YAML front matter
3. **Rebuild:** Run `npm run build:content`

## Architecture Highlights

### Data Flow
```
content/*.md → build-content.js → public/*.json → fetch() → React State → UI
```

### Component Hierarchy
```
App.tsx (state owner)
├── Header.tsx (navigation)
├── [Page].tsx (Home, Blog, Projects, Resume, Contact)
│   └── [Components] (Card, Button, Tag, etc.)
└── Footer.tsx
```

### Key Patterns

1. **No routing library** - Custom hash-based routing
2. **No global state** - Local useState hooks only
3. **Build-time content** - Markdown compiled to JSON
4. **Brutalist design** - Hard shadows, no radius, bold borders

## For AI-Assisted Development

When working on this codebase:

1. **Content changes** require running `npm run build:content`
2. **Component styling** follows brutalist pattern (see `src/index.css`)
3. **Types are centralized** in `src/types/index.ts`
4. **No tests exist** - consider adding Vitest if implementing new features
5. **Supabase is installed but unused** - planned for contact form
